The Production RAG Problem

❌ Vocabulary Mismatch

Semantic vector search often misses critical, domain-specific terminology, acronyms, product serial codes, or financial labels (e.g. "Project Aegis").

❌ Context Window Noise

Retrieving too many candidate chunks dilutes the density of relevant context, filling the context window with noise and causing model drift.

❌ Structural Ingestion Loss

Standard sentence splitters segment lists, code, and PowerPoint layout files randomly, breaking structured relationships across slide boundaries.

❌ Hallucinations & Factual Drift

Generic RAG setups encourage LLMs to synthesize answers even when the retrieved source document does not contain the required factual evidence.

Core Engineering Architecture

1. Ingestion Layer

Advanced parsers map layout semantics dynamically:

  • Page-by-page PDF extraction
  • Structural PPTX slide-boundary chunks
  • Lazy-loaded document loaders

2. Retrieval Strategy

Dense semantic understanding meets lexical keyword matching:

  • ChromaDB + all-MiniLM-L6-v2
  • Sparse BM25 Index indexing
  • Reciprocal Rank Fusion (RRF)

3. Inference & Re-ranking

High-precision filtering coupled with strict prompt governance:

  • ms-marco Cross-Encoder filter
  • Strict prompts.yaml rules
  • JSON schema validation via Pydantic
